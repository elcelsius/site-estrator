# Política de segurança

Não publique vulnerabilidades, credenciais ou dados coletados em uma issue
aberta. Use o recurso de relato privado de vulnerabilidades do GitHub quando ele
estiver habilitado no repositório.

Inclua uma descrição do impacto, a versão ou commit afetado e passos mínimos
para reprodução. Evite anexar conteúdo de terceiros que não seja necessário
para demonstrar o problema.

As versões suportadas são as publicadas na branch principal. Correções de
segurança serão priorizadas e divulgadas após existir uma mitigação adequada.

O repositório executa detecção de segredos e auditoria de dependências no
GitHub Actions. Esses controles reduzem o risco de novas exposições, mas não
substituem a rotação imediata de uma credencial comprometida.
